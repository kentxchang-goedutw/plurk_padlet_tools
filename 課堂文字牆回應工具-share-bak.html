系統設定與說明

Web 介面快速設定 Firebase

請在下方輸入框直接貼上您從 Firebase 控制台複製的 firebaseConfig 程式碼（或直接貼上含 apiKey、projectId 等六行設定），系統將自動解析並記在本機瀏覽器中，無須修改 HTML 檔案！

✓ 設定已儲存，網頁即將重載！

請依照以下步驟設定，將您的回應牆資料儲存到您自己的 Firebase 專案，實現跨裝置即時同步。

步驟一：建立 Firebase 專案

  1. 前往 Firebase 控制台。
  2. 點擊「建立專案」，並為您的專案命名（例如：my-plurk-wall）。
  3. 依照畫面指示完成專案建立，您可以選擇性地啟用 Google Analytics。

步驟二：設定 Firestore 資料庫

  1. 在左側選單中，點擊「建構」 > 「Authentication (認證)」: 點擊「開始使用」 -> 選擇「匿名」 -> 啟用並儲存。
  2. 在左側選單中，點擊「建構」 > 「Firestore Database」。
  3. 點擊「建立資料庫」。
  4. 選擇「以**測試模式**開始」。這會讓資料庫在初期可以被公開存取，方便我們測試。（重要：測試模式的存取權限會在 30 天後到期）
  5. 選擇離您最近的 Cloud Firestore 位置，然後點擊「啟用」。

圖片上傳注意

本程式會將圖片壓縮後以文字形式存入資料庫，以符合免費方案。但 Firestore 對單筆資料有 1MB 的大小限制，若您上傳的原始圖片檔案過大（例如超過 5MB 的照片），仍可能儲存失敗。若遇到此狀況，請嘗試使用尺寸較小的圖片。

步驟三：設定永久存取規則 (重要！)

測試模式會在 30 天後失效，為了讓回應牆永久可用，請手動更新安全規則：

  1. Firestore 規則:
    • 回到「Firestore Database」分頁，點擊上方的「規則」分頁。
    • 將編輯器中的內容全部取代為以下程式碼，然後點擊「發布」。
    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

安全性警告

此規則允許任何人讀取和寫入您的資料庫。這對於此公開回應牆是必要的，但請勿將任何私人或敏感資料存放在此專案中。

步驟四：取得 Firebase 設定碼並貼回本網頁

  1. 點擊左上角的專案名稱旁邊的齒輪圖示，選擇「專案設定」。
  2. 在「您的應用程式」區塊，點擊「</>」 (網頁) 圖示來註冊您的網頁應用程式。
  3. 為您的應用程式取一個暱稱，然後點擊「註冊應用程式」。
  4. 註冊後，Firebase 會顯示一個 `firebaseConfig` 物件。請點擊複製圖示，將這段完整的程式碼複製下來。
  5. 回到這個回應牆網頁，用文字編輯器打開此 HTML 檔案。
  6. 在程式碼中找到 <script type="module"> 區塊。
  7. 找到標示為「請將您從 Firebase 複製的設定貼在這裡」的區塊，用您剛剛複製的 `firebaseConfig` 物件，取代掉原本的範例內容。

const firebaseConfig = {
  apiKey: "你的firebase設定內容",
  authDomain: "你的firebase設定內容",
  projectId: "你的firebase設定內容",
  storageBucket: "你的firebase設定內容",
  messagingSenderId: "你的firebase設定內容",
  appId: "你的firebase設定內容"
};

完成！ 儲存 HTML 檔案後，重新用瀏覽器打開它。現在所有的資料都會透過您設定的 Firebase 專案進行即時同步了！

藉由本設定，您可以將互動結束後的看板資料永久備份儲存至您的 Google 試算表中，並產生一個精美、唯讀的分享連結。

步驟一：建立 Google 試算表

  1. 前往您的 Google 雲端硬碟，點擊「新增」 > 「Google 試算表」。
  2. 您可以將該試算表命名為「課堂文字牆回應存檔」。

步驟二：開啟 Apps Script 編輯器

  1. 在您新建立的試算表中，點選上方選單的「擴充功能」 > 「Apps Script」。

步驟三：複製並貼入 GAS 程式碼

  1. 切換到本說明的第三個分頁「💻 GAS 程式碼」，點擊右上角「複製程式碼」。
  2. 回到 Google Apps Script 編輯器，刪除編輯器中原本所有的內容，並將剛才複製的程式碼貼入。
  3. 點擊上方的「儲存 (磁碟片符號)」按鈕。

步驟四：部署為網頁應用程式 (Web App)

  1. 點擊右上角的「部署」 > 「新增部署」。
  2. 在彈出的視窗中，點選左上角「選取類型 (齒輪)」 > 「網頁應用程式 (Web App)」。
  3. 在「說明」欄位輸入簡短的文字，例如：`v1`。
  4. 將「誰可以存取 (Who has access)」設定為「任何人 (Anyone)」（注意：請勿選取「僅限自己」，否則外部學生網頁將無法寫入或讀取資料）。
  5. 點擊下方的「部署」按鈕。

步驟五：授權您的 Google 帳號

  1. 部署過程中會出現授權請求，請點選「授予存取權限 (Authorize access)」。
  2. 選擇您的 Google 帳戶。
  3. 若出現「Google 尚未驗證此應用程式」警告，請點選左下角的「**進階 (Advanced)**」 > 「**前往「未命名專案」 (安全起見，繼續)**」，然後點選「允許 (Allow)」。

步驟六：複製部署網址並填入網頁中

  1. 部署完成後，請複製畫面上顯示的「**網頁應用程式網址** (網址以 /exec 結尾)」。
  2. 回到此回應牆，點擊右上角的「分享 🔗」按鈕，將剛才複製的網址貼入輸入框中。系統會將此網址記在本機，以後按下分享即可自動轉存！

完成！ 轉存成功後，試算表中將會自動建立中文的工作表（看板清單、噗文清單、回應清單）與對應欄位，您可以在試算表中直接檢視或編輯全部資料，並可將產生的分享連結傳送給學生或同仁，網頁會自動以唯讀模式呈現該看板！
